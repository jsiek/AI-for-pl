# STLCSub design notes

## Types

Types are generated by

`A, B ::= Top | Nat | A -> B | { l_i : A_i }`

Record fields use numeric labels in the mechanization.  The intended
mathematical reading is by field name.

## Subtyping

The relation `A <: B` has the following clauses.

`A <: A`.

`A <: Top`.

If `B1 <: A1` and `A2 <: B2`, then `A1 -> A2 <: B1 -> B2`.

If every field `l : B` in the target record has a matching field `l : A` in the
source record with `A <: B`, then `{ ... source ... } <: { ... target ... }`.
This single record rule covers width, depth, and permutation subtyping.

## Terms and reduction

Terms are generated by variables, annotated lambdas, application, natural
constants and case analysis, record literals, and projection `M.l`.

Record literals are values. Projection reduces once the receiver is a record
literal:

`{ ..., l = M, ... }.l --> M`.

Application and natural case analysis follow the same call-by-value rules as
the nearby STLC developments.

## Typing

Typing is declarative. The subsumption rule is explicit:

If `Γ ⊢ M : A` and `A <: B`, then `Γ ⊢ M : B`.

Projection first types the receiver as a record type and then uses field
membership to select the result type.
